Fly Code

AI-Powered DevOps Workstation — Feature Overview

March 2026

Fly Code is a customized AI coding assistant built for infrastructure and DevOps workflows. It integrates directly with Jira, Jira Service Management (SECDESK), AWS, and Git to provide a unified workspace where AI agents have full context on every ticket they work on.

Fly Code main interface — sidebar with active projects, chat view, and command tray

1. Jira Ticket Import

Import Jira tickets directly into Fly Code as projects. Each imported ticket automatically:

  • Creates a dedicated project workspace
  • Generates summary.md with full ticket metadata (status, priority, components, labels, assignee)
  • Generates CLAUDE.md with ticket context, workflow rules (IaC, PRs only), and tool documentation
  • Auto-sends the ticket context to the AI agent so it immediately knows what to work on
  • Transitions the ticket to "In Progress" on Jira
  • Filters out tickets that already have active projects

Jira import dropdown — shows unresolved tickets assigned to you, filtered to exclude active projects

Jira REST API Auto-context IaC Workflow
On import, the agent receives a pre-built message with the ticket key, summary, description, and Jira URL — plus instructions to read CLAUDE.md and summary.md for full context.

2. Project Management

Active & Waiting Status

Right-click any project to toggle between Active and Waiting status. Waiting projects are visually dimmed and grouped separately in the sidebar. This status feeds into the daily standup generator.

Project context menu — toggle waiting status or remove projects

Sidebar Project Cards

Each project in the sidebar shows:

  • Ticket key and title from Jira
  • summary.md button — quick view of ticket metadata
  • SECDESK button — create or view SECDESK tickets
  • Jira link — opens the ticket directly in Jira
  • Thread history with last message preview and timestamps

Sidebar detail — SECDESK links (blue when filed), Jira links, summary.md buttons on every project

3. SECDESK Integration

Create Jira Service Management (JSM) tickets directly from any project. Used for AWS account closures, access requests, and security operations.

  • Pre-populates summary from the parent ticket
  • Selects from available JSM request types
  • After creation: appends link to summary.md, posts comment back to original Jira ticket
  • Persists SECDESK link locally — sidebar shows clickable link when a ticket exists
  • Auto-detects existing SECDESK links by scanning summary.md files
JSM API Auto-link Bi-directional
When a SECDESK ticket is filed, the sidebar button changes from "SECDESK" to a clickable blue link showing the ticket key (e.g., SECDESK-1711), alongside the original Jira ticket link.

4. Daily Standup Generator

Interactive two-step standup message builder:

  1. Select & annotate — choose which projects to include, add notes per project
  2. Edit & copy — review the generated Slack-formatted message, edit if needed, copy to clipboard

Projects are automatically grouped by Active vs Waiting/Blocked status. The generated message follows the standard format: Yesterday / Today / Blockers.

Standup modal — select projects, add notes, generate Slack-ready message

5. Command Tray

Quick-action button bar at the bottom of the chat. Each button sends a command to either the chat (for AI slash commands) or the terminal (for shell commands).

Chat view with command tray — /update, /triage, /commit, AWS SSO, Shell, Claude buttons

Customizable Chat + Terminal targets
Default buttons: /update, /triage, /commit (chat), AWS SSO (terminal — runs aws sso login), Shell, Claude. Fully editable — add, remove, or reorder buttons via the edit dialog.

6. AI Agent Context System

CLAUDE.md — Agent Instructions

Auto-generated per project with:

  • Ticket metadata (key, status, priority, type, assignee)
  • Full ticket description
  • Workflow rules: always use IaC, introduce changes through PRs
  • Tools & Environment reference documenting all available integrations

summary.md — Structured Ticket Data

Machine-readable metadata file with all Jira fields. Updated when SECDESK tickets are filed.

summary.md viewer — quick access to ticket metadata without leaving the app

Tools & Environment Reference

Embedded in every CLAUDE.md, documents:

  • Jira REST API endpoints with curl examples
  • AWS SSO profile configuration
  • GitHub CLI setup
  • SECDESK / Jira Service Management API
  • Shell, Git, Node.js, and mise environment details
